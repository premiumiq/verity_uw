Each view below is exposed at GET /api/v1/feed/{view_name} for incremental ingest into a customer-owned warehouse (Snowflake, BigQuery, Redshift, Iceberg). Always-ascending ingest_ts watermark with keyset pagination on (ingest_ts, source_pk) — same-timestamp rows never split or lose data across pages.

Pull contract
Caller passes since (ISO timestamp on first call, opaque cursor on subsequent), optional until (defaults to now), and limit (max 5000; default 1000).
Response: {"view","since","until","row_count","complete","next_since","rows"}. When complete=true the window is fully drained.
For closed-window cross-table snapshots, pass the same [since, until) to every view and wait ~5 min past until (wall clock) before treating the window as final.

Customer-side ingest recipe
# Pull v_decision incrementally into your warehouse:

since = "2020-01-01T00:00:00Z"        # first call uses ISO; later calls use the cursor
until = "2026-04-29T00:00:00Z"        # closed upper bound for the window

while True:
    r = http.get(f"{verity}/api/v1/feed/v_decision",
                 params={"since": since, "until": until, "limit": 1000})
    rows = r.json()["rows"]
    warehouse.copy_into("verity_decision", rows)   # stage + COPY into target table
    if r.json()["complete"]:
        break
    since = r.json()["next_since"]                  # opaque cursor
